# 📓 Электронный Журнал (Web-based Gradebook)

Веб-приложение для автоматизации учёта успеваемости, посещаемости и управления лабораторными работами кафедры информационных технологий.

## 🏗️ Архитектура проекта

Проект организован в виде монорепозитория:
* **`/frontend`** — Клиентское приложение на Next.js (App Router, TypeScript, Tailwind CSS, TanStack Query).
* **`/backend`** *(в разработке)* — Серверное приложение на NestJS (TypeScript, Prisma ORM, PostgreSQL, Redis, S3).

---

## 🚀 Быстрый запуск

### 1. Подготовка окружения (Инфраструктура)
Для работы базы данных PostgreSQL, кэша Redis и файлового хранилища MinIO (совместимого с AWS S3) в корне проекта настроен Docker Compose.

Запустите контейнеры:
```bash
docker-compose up -d
```

### 2. Запуск фронтенда (Next.js)
Перейдите в папку frontend, установите зависимости и запустите сервер для разработки:
```bash
cd frontend
npm install
npm run dev
```
Фронтенд будет доступен по адресу: [http://localhost:3000](http://localhost:3000)

## 📓 Функционал (Frontend 1)

* **Авторизация:** JWT-сессия (имитируется в LocalStorage для оффлайн-режима с переключением на HttpOnly Cookie). Разграничение прав доступа и защита путей (middleware).
* **Кабинет студента:**
  * Интерактивное расписание занятий на текущий день.
  * Страницы предметов с детальной успеваемостью.
  * Форма сдачи лабораторных работ (индивидуальных и командных) с drag-and-drop областью загрузки.
* **Кабинет преподавателя:**
  * Расписание проводимых пар.
  * Панель проверки присланных лабораторных с фильтрацией по статусу.
  * Выдвижная Drawer-панель для выставления оценок и комментариев студентам.

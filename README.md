# 📓 Электронный Журнал Успеваемости (Gradebook)

Веб-приложение для автоматизации учёта успеваемости, посещаемости и управления лабораторными работами. Дизайн выполнен в едином минималистичном стиле с использованием оранжевой цветовой палитры.

---

## 🏗️ Архитектура проекта

Проект организован в виде монорепозитория:
* **`/frontend`** — Клиентское приложение на Next.js (TypeScript, Tailwind CSS, Shadcn/ui).
* **`/backend`** — Серверное приложение на NestJS (TypeScript, Prisma ORM, PostgreSQL, Redis, S3).
* **`docker-compose.yml`** — Локальная инфраструктура (PostgreSQL, Redis, MinIO S3).

---

## 🚀 Быстрый запуск

### 1. Запуск инфраструктуры (Docker)
Убедитесь, что у вас запущен Docker Desktop, и выполните в корне проекта:
```bash
docker-compose up -d
```
Это поднимет PostgreSQL (порт `5432`), Redis (порт `6379`) и MinIO S3 (порт `9000`).

---

### 2. Настройка и запуск бэкенда (NestJS)
Перейдите в папку бэкенда:
```bash
cd backend
```

1. **Создание файла переменных окружения:**
   ```powershell
   # В PowerShell:
   Copy-Item .env.example .env
   
   # В командной строке CMD:
   copy .env.example .env
   ```

2. **Установка зависимостей:**
   *(Для стабильности на Windows рекомендуется флаг `--ignore-scripts`)*
   ```bash
   npm install --ignore-scripts
   ```

3. **Подготовка базы данных:**
   Сгенерируйте клиент Prisma, примените миграции и заполните БД тестовыми данными:
   ```bash
   npx prisma generate
   npx prisma migrate dev --name init
   npm run seed
   ```

4. **Запуск сервера бэкенда:**
   ```bash
   npm run start:dev
   ```
   * API будет доступно по адресу: [http://localhost:5000/api](http://localhost:5000/api)
   * Swagger-документация API: [http://localhost:5000/api/docs](http://localhost:5000/api/docs)

---

### 3. Настройка и запуск фронтенда (Next.js)
Откройте новое окно терминала и перейдите в папку фронтенда:
```bash
cd frontend
```

1. **Установка зависимостей:**
   ```bash
   npm install
   ```

2. **Запуск сервера разработчика:**
   ```bash
   npm run dev
   ```
   * Клиент будет доступен по адресу: [http://localhost:3000](http://localhost:3000)

---

## 🔗 Интеграция с живым API (Подключение к бэкенду)

По умолчанию фронтенд запущен в **Mock-режиме** (имитация API с локальной базой данных в памяти). Это позволяет полноценно тестировать функционал без развёртывания бэкенда.

Чтобы переключить фронтенд на работу с реальным NestJS сервером:
1. В файле [frontend/src/lib/api.ts](file:///d:/Desktop/Final_Js/frontend/src/lib/api.ts) установите переменную:
   ```typescript
   const USE_MOCK = false;
   ```
2. Перезапустите сервер фронтенда (`npm run dev`).

## 📓 Особенности системы оценивания
* **10-балльная шкала:** Система поддерживает оценки от `1` до `10`.
* **Быстрый клавиатурный ввод:** Нажатие клавиш `1-9` ставит оценку `1-9`, клавиша `0` ставит оценку `10`. `Backspace` / `Delete` очищает ячейку.
* **События мыши:** Правый клик отмечает пропуск (**«Н»**), клик колесиком мыши отмечает опоздание (**«О»**).
